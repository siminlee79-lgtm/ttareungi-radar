# 알림 전략

따릉이 레이더의 핵심은 "정한 시간에 저장 장소 기준 따릉이 상태를 알려주는 것"입니다.

PWA 알림은 프로토타입 단계의 참고 기능으로만 유지합니다. 실제 서비스 알림은 iOS/Android 앱 전환 후 네이티브 푸시로 구현합니다.

## 결론

- PWA: 웹 데모와 기능 검증용
- iOS/Android 앱: 실제 알림 서비스용
- 서버: 정해진 시간에 따릉이 상태를 확인하고 사용자 기기로 푸시 발송

## 왜 앱 푸시로 가는가

PWA 알림은 iPhone에서 제약이 큽니다.

- Safari에서 홈 화면에 추가해야 합니다.
- iOS 버전과 설정 영향을 많이 받습니다.
- 앱이 닫혀 있을 때 예약 알림 동작을 신뢰하기 어렵습니다.
- 앱 아이콘 숫자 배지도 안정적으로 기대하기 어렵습니다.

앱으로 전환하면 APNs와 FCM을 통해 더 안정적인 푸시를 보낼 수 있습니다.

## 목표 구조

1. 사용자가 앱에서 저장 장소 1/2와 알림 시간을 저장합니다.
2. 앱이 푸시 권한을 요청합니다.
3. iOS/Android 푸시 토큰을 서버에 저장합니다.
4. 서버가 정해진 시간에 서울시 따릉이 API를 확인합니다.
5. 서버가 마감임박/재고소진중/여유 상태와 대체 대여소를 푸시로 보냅니다.
6. 사용자가 알림을 누르면 앱의 레이더 화면으로 이동합니다.

## 기술 후보

- 앱 래핑: Capacitor
- 푸시: Firebase Cloud Messaging + Apple Push Notification service
- 서버 실행: Cloudflare Workers Cron
- 서버 DB: Cloudflare D1 또는 Supabase
- 실시간 데이터: 서울시 열린데이터광장 따릉이 API
