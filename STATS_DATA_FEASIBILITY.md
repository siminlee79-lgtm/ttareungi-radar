# 재미있는 통계 데이터 가능성 점검

기준일: 2026-06-03

## 결론

통계 카드는 삭제하지 않고 유지한다. 단, 사용자에게 보이는 방식은 다음처럼 분리한다.

- 즉시 가능: 실시간 API와 현재 저장장소 데이터로 계산
- 자동 배치 가능: 서울 열린데이터 CSV를 월 1회 내려받아 JSON으로 가공
- 자체 로그 필요: 실시간 재고 변화를 시간대별로 쌓아야 계산 가능
- 추가 외부 API 필요: 날씨처럼 따릉이 데이터 밖의 정보가 필요한 카드

## 확인된 자동 업데이트

서울 열린데이터광장 `OA-15246` 페이지는 현재 접근 가능하다.

- 데이터셋: 서울시 공공자전거 따릉이 이용정보(일별)
- 제공 정보: 대여일자, 대여소, 정기권유무, 성별, 연령대, 운동량, 이용건수
- 현재 스크립트: `scripts/fetch-latest-daily-usage.js`
- 집계 스크립트: `scripts/build-station-stats.js`
- 배포 산출물: `data/station-stats.json`

로컬 검증 결과:

- 최신 CSV 자동 다운로드 성공
- `data/station-stats.json` 집계 성공
- 원본 CSV는 `.gitignore`로 제외됨
- GitHub Actions 월간 자동 업데이트 설정 완료

## 자동 업데이트 정책

현재는 월 1회 확인으로 충분하다.

이유:

- 원본 CSV는 대용량 파일이고 실시간 데이터가 아니다.
- 서울 열린데이터 파일이 매일 갱신되지 않을 수 있다.
- GitHub Actions가 매월 최신 파일을 확인하고, 결과 JSON이 바뀐 경우만 커밋한다.

분기/반기 업데이트도 가능하지만, 앱 콘텐츠 신선도 측면에서는 월 1회가 더 낫다. 데이터가 아직 바뀌지 않았으면 자동화가 커밋하지 않으므로 불필요한 배포를 줄일 수 있다.
