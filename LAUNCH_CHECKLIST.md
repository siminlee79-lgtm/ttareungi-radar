# 따릉이 레이더 출시 준비 체크리스트

## 반영 원칙

- 지금 당장 기능 개발을 멈추고 전부 만들 단계는 아니다.
- 다만 앱 출시, 도메인 구매, 광고/수익화, 푸시 서버를 붙이기 전에는 반드시 정리한다.
- PWA 현재 단계에서는 최소한 `비공식 서비스 안내`, `데이터 출처`, `문의 이메일`, `개인정보처리방침 초안`부터 준비한다.

## 상표/브랜드 리스크 관리

### 현재 판단

- `따릉이 레이더`는 마케팅력은 좋지만, `따릉이` 상표가 포함되어 회색지대다.
- 본문에서 따릉이를 언급하는 것은 지명적 공정사용에 가까워 비교적 안전하다.
- 앱 이름, 아이콘, 로고, 색상, 문구가 서울시 공식 서비스처럼 보이면 위험하다.

### 결정 (2026-07-20)

웹·PWA·앱스토어 모두 `따릉이 레이더`로 통일한다.

`따릉이`로 검색하는 사용자가 곧 이 앱의 사용자인데, 영문명으로는 그 검색에
잡히지 않아 유입이 사실상 끊긴다. 아래 금지 표현대로 `따릉이` 단독 앱명은
피하되, `따릉이 + 레이더` 조합은 허용 범위로 본다.

공식 오인을 막는 장치는 함께 간다.

- 짧은 설명 첫머리에 `비공식`을 명시한다.
- 긴 설명에 서울시 공식 앱이 아님을 한 문장으로 못박는다.
- 아이콘 배경은 남색을 쓴다. 따릉이 공식 브랜드가 초록 계열이므로 초록을
  주색으로 깔지 않는다.

### 금지 표현

- 공식
- official
- 서울시 공식
- 서울시설공단 공식
- 따릉이 단독 앱명
- 서울시/따릉이 공식 로고 오인 가능 디자인

## 필수 문구

푸터, 설정, 약관 페이지 중 최소 한 곳에 아래 내용을 넣는다.

> 본 서비스는 서울시, 서울시설공단, 서울자전거 따릉이의 공식 서비스가 아니며, 서울 열린데이터광장에서 제공하는 공공 API를 활용한 비공식 정보 제공 서비스입니다.

> 본 서비스가 제공하는 대여소 정보는 공공 API를 기반으로 하며, 실제 현장 상황과 일시적 차이가 있을 수 있습니다.

> 데이터 출처: 서울 열린데이터광장(data.seoul.go.kr)

## 필수 페이지

앱 출시 전 필수:

- 개인정보처리방침
- 서비스 이용약관
- 위치정보 이용약관
- 운영자 정보
- 문의하기
- 데이터 출처 안내

PWA 현재 단계에서 먼저 만들 것:

- 개인정보처리방침 초안
- 푸터 면책 문구
- 데이터 출처 표기
- 문의 이메일 표기

## 개인정보처리방침 포함 항목

- 수집 항목: 위치정보, 저장 장소, 알림 시간, 디바이스 푸시 토큰, 사용 로그
- 이용 목적: 주변 대여소 검색, 저장 장소 알림, 서비스 개선
- 보유 기간: 사용자가 삭제하거나 서비스 탈퇴/철회할 때까지
- 제3자 제공 여부
- 위탁/외부 도구: Firebase, GA4, Cloudflare, Supabase 등 사용 시 명시
- 정보주체 권리 행사 방법
- 개인정보 보호책임자와 문의 이메일
- 변경 고지 방법

## 위치정보 관련

- 현재 PWA의 브라우저 위치 사용도 위치정보 안내가 필요하다.
- 서버에 저장 장소/푸시 알림을 붙이면 위치기반서비스사업 신고 여부를 검토한다.
- 앱 출시 전 위치정보 이용약관을 별도로 둔다.

## 수익화 전 체크

- AdSense/AdMob 적용 전 개인정보처리방침 업데이트
- 광고 식별자/분석 도구 사용 여부 고지
- 유료 구독 도입 시 통신판매업 신고 검토
- 인앱결제 정책 확인

## 우선순위

1. PWA 기능 안정화
2. 푸터에 비공식 서비스/데이터 출처 문구 추가
3. 개인정보처리방침 초안 페이지 추가
4. 도메인 구매 여부 결정
5. 위치정보 이용약관/서비스 이용약관 추가
6. 푸시 서버/DB 붙이기 전 법적 문구 재검토
7. 앱스토어 출시 전 앱 이름과 아이콘 최종 점검
